# 청소 플랫폼 설계도
> 투명성 기반 청소 서비스 중개 플랫폼 (웹 → 앱 전환 목표)

---

## 1. 기술 스택

### Frontend
| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | React 18 | 현재 프로젝트 기준, 나중에 React Native 전환 유리 |
| 스타일 | Tailwind CSS | 모바일 퍼스트, 빠른 UI 구현 |
| 라우팅 | React Router v6 | 페이지 전환 |
| 상태관리 | Zustand | Redux보다 가볍고 간단 |
| 지도/위치 | Kakao Map API | 국내 주소 정확도 최고 |
| HTTP | Axios | API 통신 |
| 실시간 채팅 | SockJS + STOMP | WebSocket 기반 1:1 메시지 |
| 아이콘 | React Icons | 다양한 아이콘 세트 |

### Backend
| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Spring Boot 3 | 현재 프로젝트 기준 |
| DB | MySQL | 관계형 데이터 구조에 적합 |
| ORM | JPA + Hibernate | 객체-DB 매핑 |
| 인증 | JWT (AccessToken + RefreshToken) | Stateless 인증 |
| 실시간 | WebSocket (STOMP) | 1:1 채팅 |
| 파일 업로드 | AWS S3 또는 로컬 스토리지 | 프로필/리뷰 사진 |
| 빌드 | Gradle | Spring Boot 표준 |

### 배포 (나중에)
- Frontend: Vercel 또는 Netlify
- Backend: AWS EC2 또는 Railway
- DB: AWS RDS (MySQL)

## 3. DB 스키마 (ERD)

### 테이블 목록

#### customers (고객)
```
id              BIGINT PK AUTO_INCREMENT
name            VARCHAR(50)       이름
email           VARCHAR(100) UNIQUE
password        VARCHAR(255)      암호화
phone           VARCHAR(20)
address         VARCHAR(200)      기본 주소
created_at      DATETIME
```

#### companies (청소 업체)
```
id              BIGINT PK AUTO_INCREMENT
name            VARCHAR(100)      업체명
email           VARCHAR(100) UNIQUE
password        VARCHAR(255)
phone           VARCHAR(20)
representative  VARCHAR(50)       대표자명
business_no     VARCHAR(20)       사업자번호
description     TEXT              업체 소개
profile_image   VARCHAR(500)      프로필 사진 URL
sido            VARCHAR(20)       시/도
sigungu         VARCHAR(20)       시/군/구
dong            VARCHAR(20)       동
address_detail  VARCHAR(200)      상세 주소
latitude        DOUBLE            위도 (GPS용)
longitude       DOUBLE            경도 (GPS용)
avg_rating      DECIMAL(2,1)      평균 별점 (자동 갱신)
review_count    INT DEFAULT 0     리뷰 수 (자동 갱신)
is_verified     BOOLEAN           사업자 인증 여부
is_active       BOOLEAN           서비스 활성화 여부
created_at      DATETIME
```

#### cleaning_options (업체별 서비스 항목)
```
id              BIGINT PK
company_id      BIGINT FK → companies.id
option_name     VARCHAR(100)      예) 가정집 기본청소, 입주청소
price_min       INT               최소 금액
price_max       INT               최대 금액
unit            VARCHAR(20)       단위 (예: 평당, 1회)
description     TEXT              설명
```

#### cleaning_requests (고객 의뢰)
```
id              BIGINT PK
customer_id     BIGINT FK → customers.id
company_id      BIGINT FK → companies.id
option_id       BIGINT FK → cleaning_options.id
request_address VARCHAR(300)      청소 장소 주소
area_size       INT               평수
request_date    DATE              희망 날짜
request_time    VARCHAR(20)       희망 시간대
memo            TEXT              요청 메모
status          ENUM('PENDING','ACCEPTED','REJECTED','IN_PROGRESS','COMPLETED','CANCELLED')
created_at      DATETIME
```

#### quotations (견적서)
```
id              BIGINT PK
request_id      BIGINT FK → cleaning_requests.id
company_id      BIGINT FK → companies.id
total_price     INT               견적 총액
detail          TEXT              견적 상세 내용
visit_date      DATE              방문 예정일
visit_time      VARCHAR(20)       방문 예정 시간
valid_until     DATE              견적 유효기간
status          ENUM('SENT','ACCEPTED','REJECTED')
created_at      DATETIME
```

#### reviews (리뷰)
```
id              BIGINT PK
customer_id     BIGINT FK → customers.id
company_id      BIGINT FK → companies.id
request_id      BIGINT FK → cleaning_requests.id
rating          TINYINT           1~5 별점
content         TEXT              리뷰 내용
before_image    VARCHAR(500)      청소 전 사진
after_image     VARCHAR(500)      청소 후 사진
created_at      DATETIME
```

#### chat_rooms (채팅방)
```
id              BIGINT PK
customer_id     BIGINT FK → customers.id
company_id      BIGINT FK → companies.id
request_id      BIGINT FK → cleaning_requests.id (nullable)
created_at      DATETIME
```

#### chat_messages (채팅 메시지)
```
id              BIGINT PK
room_id         BIGINT FK → chat_rooms.id
sender_type     ENUM('CUSTOMER','COMPANY')
sender_id       BIGINT
content         TEXT
is_read         BOOLEAN DEFAULT false
created_at      DATETIME
```

## 4. 사용자 흐름도 (Flow Chart)

### [공통] 앱 진입
```
앱 실행
    │
    ├─ 로그인 상태? ─── NO ──→ 홈 화면 (비로그인)
    │                              │
    │                         위치 허용 팝업
    │                         ┌───┴───┐
    │                       허용     미허용
    │                         │         │
    │                     GPS 위치    지역 직접 입력
    │                         │    (시/도 → 시군구 → 동)
    │                         └───┬───┘
    │                         업체 목록 (별점+리뷰순)
    │
    └─ 로그인 상태? ─── YES ─→ 역할 확인
                                ┌───┴───┐
                             고객      업체
                                │         │
                            고객 홈    업체 대시보드
```

### [고객] 회원가입
```
회원가입 선택
    │
    └→ "고객으로 가입" 선택
            │
        이름 / 이메일 / 비밀번호 / 전화번호 입력
            │
        이메일 인증
            │
        가입 완료 → 로그인 → 고객 홈
```

### [업체] 회원가입
```
회원가입 선택
    │
    └→ "청소 업체로 가입" 선택
            │
        업체명 / 대표자 / 사업자번호 / 이메일 / 비밀번호 / 전화번호 입력
            │
        업체 주소 입력 (시/도/구/동/상세)
            │
        서비스 항목 등록 (가정청소, 입주청소 등 + 가격)
            │
        프로필 사진 업로드
            │
        사업자등록증 업로드 (관리자 인증 대기)
            │
        가입 완료 (관리자 승인 후 활성화)
```

### [고객] 업체 검색 → 의뢰 흐름
```
홈 화면
    │
    ├─ GPS 허용 → 현재 위치 기반 업체 목록
    └─ GPS 미허용 → 시/도 선택 → 시군구 선택 → 동 선택 → 업체 목록
                                                            │
                                                    정렬 옵션
                                               (별점순 / 리뷰많은순 / 거리순)
                                                            │
                                                    업체 카드 목록
                                                (업체명, 별점, 리뷰수, 서비스태그)
                                                            │
                                                    업체 상세 클릭
                                                            │
                                            ┌───────────────┤
                                     업체 정보              리뷰 목록
                                     서비스 항목            청소 전/후 사진
                                     가격 범위              평균 별점
                                            │
                                        "의뢰하기" 버튼
                                            │
                                    로그인 확인
                                    ┌───┴───┐
                                 미로그인   로그인
                                    │         │
                               로그인 유도   의뢰 작성 화면
                                                │
                                        청소 장소 주소 입력
                                        서비스 항목 선택
                                        평수 입력
                                        희망 날짜/시간 선택
                                        요청 메모 입력
                                                │
                                        의뢰 전송 → 업체에 알림
                                                │
                                        채팅방 자동 생성
                                                │
                                        1:1 채팅으로 연결
```

### [업체] 의뢰 수신 → 견적서 흐름
```
업체 대시보드
    │
    └→ 새 의뢰 알림
            │
        의뢰 상세 확인
        (고객 주소, 평수, 날짜, 요청사항)
            │
        ┌───┴───┐
      수락      거절
        │         │
    견적서 작성  거절 사유 입력
        │         │
    총 금액      고객에게 알림
    방문 예정일
    세부 내역
    유효기간
        │
    견적서 발송 → 고객에게 알림
        │
    고객이 견적 확인
    ┌───┴───┐
  수락      거절
    │         │
예약 확정   업체에 알림
    │
실시간 상태 업데이트
(예약확정 → 진행중 → 완료)
    │
완료 후 리뷰 작성 유도 (고객)
```

### [공통] 1:1 채팅 흐름
```
채팅 진입 (의뢰 페이지 or 채팅 목록)
    │
WebSocket 연결
    │
메시지 송수신 (실시간)
    │
    ├─ 텍스트 메시지
    ├─ 사진 첨부
    └─ 견적서 링크 공유
```

## 6. API 엔드포인트 설계

### 인증 (Auth)
```
POST   /api/auth/customer/signup     고객 회원가입
POST   /api/auth/company/signup      업체 회원가입
POST   /api/auth/login               로그인 (고객/업체 공통)
POST   /api/auth/logout              로그아웃
POST   /api/auth/refresh             토큰 갱신
```

### 업체 (Company)
```
GET    /api/companies                업체 목록 (지역/GPS 기반)
       ?sido=서울&sigungu=강남구
       ?lat=37.5&lng=127.0&radius=3
       ?sort=rating | reviews | distance

GET    /api/companies/{id}           업체 상세 조회
PUT    /api/companies/{id}           업체 정보 수정 (본인만)
GET    /api/companies/{id}/reviews   업체 리뷰 목록
GET    /api/companies/{id}/options   업체 서비스 항목
POST   /api/companies/{id}/options   서비스 항목 추가
```

### 의뢰 (Request)
```
POST   /api/requests                 의뢰 보내기 (고객)
GET    /api/requests/customer        내 의뢰 목록 (고객)
GET    /api/requests/company         받은 의뢰 목록 (업체)
GET    /api/requests/{id}            의뢰 상세
PATCH  /api/requests/{id}/status     의뢰 상태 변경 (업체)
```

### 견적서 (Quotation)
```
POST   /api/quotations               견적서 발송 (업체)
GET    /api/quotations/{id}          견적서 조회
PATCH  /api/quotations/{id}/accept   견적 수락 (고객)
PATCH  /api/quotations/{id}/reject   견적 거절 (고객)
```

### 리뷰 (Review)
```
POST   /api/reviews                  리뷰 작성 (고객, 완료 후만)
GET    /api/reviews/company/{id}     업체 리뷰 목록
```

### 채팅 (Chat)
```
GET    /api/chats                    채팅방 목록
GET    /api/chats/{roomId}/messages  메시지 내역
POST   /api/chats/{requestId}        채팅방 생성 (의뢰 연결)

WS     /ws/chat                      WebSocket 연결
STOMP  /pub/chat/{roomId}            메시지 발송
STOMP  /sub/chat/{roomId}            메시지 수신
```

### 지역 데이터
```
GET    /api/regions/sido             시/도 목록
GET    /api/regions/sigungu?sido=    시/군/구 목록
GET    /api/regions/dong?sigungu=    동/읍/면 목록
```

## 7. 주요 기능 상세 설계

### GPS 위치 기반 검색
```javascript
// 위치 허용 시
navigator.geolocation.getCurrentPosition(
  (position) => {
    const { latitude, longitude } = position.coords;
    // 위도/경도로 주변 업체 검색 (반경 3km 기본)
    fetchCompanies({ lat: latitude, lng: longitude, radius: 3 });
  },
  () => {
    // 미허용 시 지역 선택 UI 표시
    showRegionSelector();
  }
);
```

### 업체 정렬 기준
- **별점순**: avg_rating DESC, review_count DESC
- **리뷰많은순**: review_count DESC, avg_rating DESC
- **거리순**: GPS 허용 시만 활성화, 반경 내 거리 계산

### 투명성 핵심 기능
1. **청소 전/후 사진** — 완료 시 업체가 필수 업로드
2. **항목별 견적** — 숨은 비용 없이 세부 내역 공개
3. **실시간 상태 추적** — 예약확정 → 진행중 → 완료
4. **검증된 리뷰** — 실제 완료된 의뢰에만 리뷰 작성 가능
5. **사업자 인증 뱃지** — 인증된 업체만 뱃지 표시

## 9. 나중에 앱 전환 시 (React Native)

웹을 React + Tailwind로 만들면 앱 전환 시:
- **비즈니스 로직** (API 호출, 상태관리) → 거의 그대로 재사용
- **컴포넌트** → JSX 구조 참고해서 RN 컴포넌트로 교체
- **스타일** → Tailwind → StyleSheet로 변환
- **위치** → `navigator.geolocation` → `expo-location`으로 교체
- **채팅** → WebSocket 로직 그대로 재사용 가능

> 핵심 화면 구조와 API는 100% 공유됩니다.
